# HOOK-INTEGRATION — Claude Code Hook Spec

> Highest-risk component. Read this before building anything in `src/hook.ts`.

---

## What the Hook Does

When a Claude Code session approaches its context limit, PACT intercepts, compresses
the conversation history into PACT syntax, and injects the compressed version as a
system message. The model carries 35x fewer tokens without losing task state.

---

## Hook Event: `PreToolUse`

Claude Code does not expose a dedicated `context-approaching-limit` event. The
practical trigger is `PreToolUse`, which fires before every tool call — giving us
a reliable checkpoint to measure and compress.

**Why PreToolUse:**
- Fires on every tool invocation (Bash, Read, Edit, etc.)
- Runs synchronously — can block the tool call if needed
- Hook receives tool metadata via stdin
- Hook response can inject context via `prompt_inject`

---

## Hook Registration

`npx pact-cc install` writes this entry into the project's `.claude/settings.json`:

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",
            "command": "node .pact/hook.js"
          }
        ]
      }
    ]
  }
}
```

If `.claude/settings.json` already exists, `pact-cc install` merges — it does not
overwrite. It reads the file, appends to the `PreToolUse` array, and writes back.

---

## Hook Script: `.pact/hook.js`

Auto-generated by `pact-cc install`. Never hand-edited. Regenerate with `pact-cc install`.

```js
// .pact/hook.js — auto-generated by pact-cc install
import { readFileSync, writeFileSync, existsSync } from 'fs'
import { runPACT } from '../pact-engine.js'

const THRESHOLD = 0.80  // compress when context is >= 80% of max tokens
const STATE_FILE = '.pact/state.json'

async function main() {
  let hookInput
  try {
    hookInput = JSON.parse(readFileSync('/dev/stdin', 'utf8'))
  } catch {
    respond({ continue: true })
    return
  }

  const { context_tokens, max_tokens } = hookInput
  const usage = context_tokens / max_tokens

  if (usage < THRESHOLD) {
    respond({ continue: true })
    return
  }

  // Compress
  const state = loadState()
  const compressed = await compress(hookInput.conversation_history, state)

  if (!compressed) {
    respond({ continue: true })  // never block on compression error
    return
  }

  saveState(compressed.state)
  respond({
    continue: true,
    prompt_inject: compressed.pact_summary
  })
}

function respond(obj) {
  process.stdout.write(JSON.stringify(obj) + '\n')
}

function loadState() {
  if (existsSync(STATE_FILE)) {
    try { return JSON.parse(readFileSync(STATE_FILE, 'utf8')) } catch { return {} }
  }
  return {}
}

function saveState(state) {
  writeFileSync(STATE_FILE, JSON.stringify(state, null, 2))
}

main().catch(() => respond({ continue: true }))
```

## Compression Pipeline (hook.ts → compress.ts)

```
conversation_history (natural language)
         │
         ▼
  compress.ts: buildCompressionPrompt(history, existingState)
         │
         ▼
  Claude API call (claude-haiku-4-5, fast/cheap)
  System: "You are a PACT encoder. Convert the agent context below to PACT syntax."
  User: [conversation history]
         │
         ▼
  Claude returns PACT program string
         │
         ▼
  pact-engine.js: runPACT(pactString) — validate
         │  (if error → return null, keep original)
         ▼
  { pact_summary: string, state: object }
```

**Model choice for compression calls:** `claude-haiku-4-5-20251001` — fast, cheap,
good enough for structure extraction. Reserve Opus for the actual agent work.

## Compression System Prompt

`src/compress.ts` sends this system prompt to Claude for compression:

```
You are a PACT encoder. Your job is to compress an AI agent's conversation history
into PACT syntax — a compact scripting language that preserves all task-critical
information.

PACT syntax rules:
- Variables: name = value
- Maps: { key: value, key2: value2 }
- Arrays: [item1, item2, item3]
- Print: . expression
- Functions: fn(arg1, arg2)
- Strings: 'text here'

Encode the agent's current state as a PACT program that outputs a JSON object
with these fields:
- goal: the current task goal (string)
- files: map of file path → { reads, writes, state: 'done'|'in_progress'|'pending' }
- plan_done: completed steps (array of strings)
- plan_next: remaining steps (array of strings)
- constraints: hard constraints that must not be violated (array of strings)
- entities: named entities with their current state

Output ONLY the PACT program. No explanation. No markdown fences.
The program must end with `. sjson(session)` to output the final state.
```

## Rehydration

When the agent needs full context on a specific entity, `rehydrate.ts` expands
the relevant PACT subgraph back to natural language. This is the "lossless"
part of the system — we only expand what's currently needed.

**Rehydration trigger:** The hook checks if the injected PACT summary is the
active context. If the agent's next tool call references an entity not in the
expanded form, rehydrate it first.

**Rehydration prompt** (sent to Claude Haiku):
```
The following PACT program encodes agent state for a coding session.
Expand the '{entity}' section into natural language that an AI assistant
can reason over. Be specific and complete. Output only the expanded text.

PACT:
{pact_program}
```

---

## Hook Response Contract

```typescript
type HookResponse = {
  continue: true            // always true — never block the tool call
  prompt_inject?: string    // optional: PACT summary to inject as system context
}
```

**Invariant: never return `continue: false`.** If compression fails, fall through
silently. The user's Claude Code session must never be interrupted by PACT errors.

---

## State File: `.pact/state.json`

Persists compressed state across tool calls within a session.

```json
{
  "session_id": "abc123",
  "compressed_at_tokens": 145000,
  "compression_ratio": 34.7,
  "pact": "session = { goal: '...', files: {...} }\n. sjson(session)",
  "entity_cache": {
    "UserAuth": "...",
    "SessionToken": "..."
  }
}
```

Invalidated on: `pact-cc uninstall`, new session start, or when `session_id` changes.
Gitignored — never committed.

---

## Error Handling Matrix

| Failure | Behavior |
|---------|----------|
| Claude API call fails | `continue: true`, no injection, log to stderr |
| PACT validation fails | `continue: true`, no injection, keep original context |
| State file corrupt | Reset state to `{}`, continue |
| Node crashes in hook | Claude Code sees non-zero exit — falls through to tool call |
| Hook takes > 5s | Claude Code timeout — falls through, tool call proceeds |

**Principle:** PACT is a middleware enhancement. Failures must be invisible to the user.

---

## `npx pact-cc install` — What it does

1. Check if `.claude/` dir exists; create if not
2. Read `.claude/settings.json` (or start with `{}`)
3. Append hook entry to `PreToolUse` array (if not already present)
4. Write `.claude/settings.json`
5. Create `.pact/` directory
6. Write `.pact/hook.js` (auto-generated hook script)
7. Print: `✓ PACT installed. Token compression active on next session.`

## `npx pact-cc uninstall` — What it does

1. Read `.claude/settings.json`
2. Remove PACT hook entry from `PreToolUse` array
3. Write back `.claude/settings.json`
4. Delete `.pact/` directory and contents
5. Print: `✓ PACT uninstalled.`
